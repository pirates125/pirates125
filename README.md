## So, What do I do ?

My goal in every project I develop is not to have code that screams and looks like mud, but rather to have a guide that speaks, explains the logic, shows the picture and illuminates everything. My working principle is as follows;

- Everything should be clear, simple, and understandable.
- Code should be able to express itself without shouting.
- If a structure doesn't flow flawlessly, it doesn't work.
- Errors aren't hidden; they are made visible and managed.
- To understand a system, its flows must be known in advance.
- Unnecessary complexity is the enemy of good design.
- Software is the expression of the mind; the more organized the thought, the more organized the code.
